# InternLAB

InternLAB - монолитное Django-приложение для интеллектуального подбора стажировок студентам и подбора кандидатов работодателям.

Проект выполнен в учебном формате (диплом / ВКР), поэтому основной фокус:
- понятная архитектура,
- прозрачная бизнес-логика,
- простая поддержка и объяснимость на защите.

## Описание проекта
Платформа позволяет:
- студентам заполнять профиль и навыки, просматривать подходящие вакансии и отправлять заявки;
- работодателям вести профиль компании, публиковать вакансии, задавать требования по навыкам и рассматривать заявки;
- системе рассчитывать степень совпадения студента и вакансии по взвешенной формуле.

## Технологический стек
- Python 3.11+
- Django 4.2
- PostgreSQL (основная БД)
- SQLite (локальный fallback для быстрого запуска)
- Django ORM
- Django Templates (server-rendered)
- Bootstrap 5
- Django Admin

## Краткий обзор архитектуры
Архитектура монолитная:

`Пользователь -> Django Views -> Services/бизнес-логика -> Models (ORM) -> База данных`

Особенности:
- без SPA;
- без React;
- без микросервисов;
- ключевая логика (matching) вынесена в сервисный слой.

- управление навыками;
- просмотр вакансий;
- получение рекомендаций;
- отправка заявок;

### Работодатель
- регистрация и вход;
- создание/редактирование профиля компании;

- управление требуемыми навыками вакансий;

## Основные функции
- аутентификация и роль-доступ;
- профили студентов и работодателей;
- каталог навыков и уровней;
- вакансии и их статусы (`draft`, `published`, `closed`, `archived`);
- требуемые навыки вакансий (`required_level`, `weight`, `is_critical`);
- заявки студентов;
- модуль рекомендаций и расчет совпадения.

## Логика сопоставления
Логика реализована в:
- `apps/matching/services.py`

Для каждой опубликованной вакансии:
1. Берутся требуемые навыки вакансии.
2. Сравниваются с навыками студента.
3. Для каждого навыка считается:
   - `skill_match = min(student_level / required_level, 1)`
4. Итоговая оценка:
   - `score = sum(skill_match * weight) / sum(weight) * 100`
5. Если отсутствует хотя бы один критичный навык, применяется штраф:
   - `score = score * 0.8`
6. Возвращаются:
   - вакансия,
   - процент совпадения,
   - список недостающих навыков,
   - флаг отсутствия критичных навыков.
7. Рекомендации сортируются по убыванию `score`.

## Локальный запуск
1. Создать и активировать виртуальное окружение.
2. Установить зависимости:
   - `pip install -r requirements.txt`
3. Выполнить миграции:
   - `python manage.py migrate`
4. Создать суперпользователя:
   - `python manage.py createsuperuser`
5. (Опционально) загрузить демо-данные:
   - `python manage.py seed_demo_data --reset`
6. Запустить сервер:
   - `python manage.py runserver`
7. Открыть:
   - `http://127.0.0.1:8000/`

## Переменные окружения
Для PostgreSQL укажите:
- `POSTGRES_DB`
- `POSTGRES_USER`
- `POSTGRES_PASSWORD`
- `POSTGRES_HOST`
- `POSTGRES_PORT`

Если переменные не заданы, проект автоматически запускается на SQLite (`db.sqlite3`).
